1 CKL_PSM

介绍

本版本ckl_psm使用typescript进行开发，node进行前端服务部署。其分工如下：

1.后端服务器：负责存储训练好的PCFG相关模型，并传输给客户端。

2.客户端：负责接收服务器传输的PCFG模型，并使用该模型的相关参数，在本地对输入口令进行安全度检测。

 

客户端打开时，首先进行一次初始化行为，调用load_model()与load_ranks()接口，向服务器请求训练好的加密PCFG模型并存储在本地。后续所有行为均在客户端本地完成，使用该模型参数对输入口令在本地进行安全度检测。

1.1 项目配置需求

语言：typescript+python

需求：

python3.6或python3.8

Node14.17及yarn1.22（前端服务部署）

系统：Ubuntu20.04或Windows10

1.2 部署方法

后端资源服务器：

xxxxxxxxxx
cd backend
pip3 install -r requirements
python3 pcfg_server.py  # The default ip:port is <device local ip>:3001, and it MUSE BE <device local ip>:3001

前端：

xxxxxxxxxx
cd frontend
yarn install
python3 ipconfig.py  # it is the same as: echo <device local ip> > ./src/ip.json
yarn build
yarn global add serve
~/.yarn/bin/serve build  # It will automatically choose a port
1.3 项目文件树

项目中主要分为frontend、backend两个文件夹。backend主要负责资源服务器的部署。而frontend中，相关的前端api封装在App.tsx中。

3.本地度量口令安全度

接口名

load_model()

接口功能

使用已经存储在本地的蒙特卡罗表与PCFG模型，对口令安全度进行度量，返回相应的度量数据。其中，口令将会被切分成数个口令片段。输出内容包括了口令整体的量化评估结果，以及每个片段的危险性。

错误情况
